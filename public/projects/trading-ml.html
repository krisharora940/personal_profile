Overview

What This Project Is

This is a systematic exploration of machine learning applications to trading, with rigorous backtesting, validation procedures, and performance tracking. Rather than attempting prediction theater, this project focuses on disciplined hypothesis testing and understanding edge through careful evaluation.

The research emphasizes reproducibility, anti-leakage validation, and honest documentation of both successful and failed experiments.

Research Question

Can disciplined ML experimentation identify trading signals with positive expectancy in intraday equity index futures, when validated against walk-forward data and subjected to rigorous anti-leakage controls?

This shifts the focus from "predicting the market" to "identifying consistent, repeatable patterns under rigorous validation."

Hypothesis 1

Volume and price action patterns can provide edge in 5-minute timeframes.

Hypothesis 2

Feature engineering from order flow imbalance improves model accuracy.

Hypothesis 3

Ensemble models outperform single models under walk-forward validation.

Validation and Risk Controls

Walk-Forward Analysis

The model is trained on historical data and tested on subsequent unseen data in sequential, non-overlapping windows. This prevents look-ahead bias and provides realistic out-of-sample performance.

Training: 3 months | Testing: 1 month | Rolling window every month

Anti-Leakage Measures

✓ No future data used in training
✓ Features computed using only past data
✓ Indicators re-calculated on unseen periods
✓ Separate scaling per fold

Risk Management

✓ Fixed position sizing: 1 contract per signal
✓ Stop-loss: 2 ATR below entry
✓ Take-profit: Risk-reward ratio 1:2
✓ Max 5 concurrent positions

Lessons and Next Iterations

What Worked

  • Walk-forward validation

    Prevented overfitting and provided realistic expectations

  • Volume-based features

    Order imbalance proved more predictive than price patterns alone

  • Risk management discipline

    Fixed stops limited drawdown effectively

Limitations & Future Work

  • Limited to single instrument

    Need to test across more contracts and asset classes

  • Real-time slippage

    Live performance differs from backtest; need realistic slippage modeling

  • Market regime changes

    Developing adaptive models for changing volatility environments

Next Iterations

  • → Expand to multiple instruments (NQ, YM, GC)
  • → Implement adaptive position sizing based on volatility
  • → Add macro regime filters (VIX, yield curve)
  • → Paper trade live model to validate real-time performance
